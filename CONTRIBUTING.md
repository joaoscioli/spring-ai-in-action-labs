# Contributing

This repository is maintained as a Spring AI engineering lab.

## Working Style

- Add one AI capability or evaluation concept at a time.
- Keep examples useful for backend product features.
- Avoid committing secrets, real API keys, or sensitive prompts.
- Document cost, latency, reliability, and safety trade-offs.

## Before Opening A Pull Request

- Run the test suite.
- Check whether prompt, evaluation, or safety documentation needs an update.
- Confirm that examples can run without leaking credentials.

## Review Focus

Reviewers should look for practical AI integration, testability, observability,
safe configuration, and clear business value.
